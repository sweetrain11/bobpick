## 2. 개요 (Overview)

### 2.1 제품 요약
키워드 기반으로 음식점을 검색하고, 사용자가 직접 구성한 후보 메뉴를 대상으로 참여형 랜덤 게임(사다리타기, 제비뽑기, 복권 긁기, 룰렛, 슬롯머신)을 통해 메뉴를 결정하도록 돕는 웹 서비스.

### 2.2 문제 정의 (Problem Statement)
- 사용자는 "무엇을 먹을지" 결정하는 데 반복적인 인지 부담을 느낀다.
- 기존 음식점 검색 서비스는 탐색은 지원하지만 최종 결정 단계를 지원하지 않는다.
- 단순 랜덤 추천은 사용자의 선호를 반영하지 못해 결과 수용도가 낮다.

### 2.3 솔루션 핵심 가설
- 사용자가 후보 메뉴를 직접 구성(추가/제외)하면 랜덤 결과에 대한 수용도가 높아진다.
- 결과 도출 과정을 게임화하면 결정 과정 자체가 즐거운 경험이 되어 재방문을 유도한다.

---

## 3. 목표 및 성공 지표 (Goals & Metrics)

### 3.1 제품 목표
1. 키워드 검색으로 원하는 음식점을 쉽고 빠르게 찾을 수 있게 한다.
2. 메뉴 결정 부담을 줄이고, 결정 과정을 재미있는 경험으로 전환한다.
3. 사용자가 게임에 직접 참여하여 메뉴를 결정하는 인터랙티브 경험을 제공한다.

### 3.2 성공 지표 (KPI, 제안)
| 지표 | 정의 | 목표(초기) |
|---|---|---|
| 게임 완료율 | 게임 시작 대비 결과 확인 비율 | ≥ 70% |
| 검색→게임 전환율 | 검색 사용자 중 게임 진입 비율 | ≥ 30% |
| 재게임률 | 결과 확인 후 다시 게임을 돌리는 비율 | 모니터링 |
| 재방문율 | 7일 내 재방문 비율 | 모니터링 |

## 4. 사용자 정의 (Target Users)

### 4.1 주요 페르소나
- **점심 결정 장애 직장인:** 매일 반복되는 점심 메뉴 고민을 빠르고 재미있게 해결하고 싶음.
- **모임 주도자:** 친구/동료와 함께 메뉴를 공정하고 재미있게 정하고 싶음.
- **탐색형 사용자:** 새로운 음식점을 찾아보다가 메뉴 선택까지 한 번에 끝내고 싶음.

### 4.2 사용자 스토리
- 사용자는 키워드로 음식점을 검색하여 마음에 드는 곳을 찾을 수 있다.
- 사용자는 음식점의 전체 메뉴를 확인하고 후보 메뉴를 직접 넣거나 뺄 수 있다.
- 사용자는 5가지 랜덤 게임 중 하나를 골라 참여하고, 선택된 메뉴 결과를 확인할 수 있다.

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 키워드 기반 음식점 검색 [P0]
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-1.1 | 사용자는 검색창에 키워드를 입력하여 음식점을 검색할 수 있다 | P0 |
| FR-1.2 | 검색 결과는 음식점 목록 형태로 표시된다 (상호명, 대표 정보 포함) | P0 |
| FR-1.3 | 검색 결과에서 특정 음식점을 선택하여 상세(메뉴)로 진입할 수 있다 | P0 |
| FR-1.4 | 검색 결과가 없는 경우 안내 메시지를 표시한다 | P1 |

### 5.2 게임 전 후보 메뉴 구성 [P0]
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-2.1 | 선택한 음식점의 전체 메뉴 목록을 표시한다 | P0 |
| FR-2.2 | 사용자는 개별 메뉴를 후보 목록에 추가/제외할 수 있다 | P0 |
| FR-2.3 | 현재 후보 목록을 실시간으로 확인할 수 있다 | P0 |
| FR-2.4 | 후보 메뉴가 최소 2개 이상일 때만 게임을 시작할 수 있다 | P0 |
| FR-2.5 | 전체 선택/전체 해제 기능을 제공한다 | P1 |

### 5.3 랜덤 게임형 메뉴 선택 [P0]
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-3.1 | 사용자는 5가지 게임(사다리타기, 제비뽑기, 복권 긁기, 룰렛, 슬롯머신) 중 하나를 선택할 수 있다 | P0 |
| FR-3.2 | 각 게임은 후보 메뉴 중 하나를 공정한 랜덤 방식으로 선택한다 | P0 |
| FR-3.3 | 게임은 사용자 인터랙션(긁기, 뽑기, 돌리기 등)을 포함한 참여형으로 구현한다 | P0 |
| FR-3.4 | 게임 종료 후 선택된 메뉴를 결과 화면에 표시한다 | P0 |
| FR-3.5 | 결과 화면에서 "다시 하기" 및 "다른 게임으로 하기"를 제공한다 | P1 |

#### 게임별 상세
1. **사다리타기:** 후보 메뉴 수만큼 세로줄 생성, 사용자가 출발점 선택 → 경로 애니메이션 → 도착 메뉴 표시
2. **제비뽑기:** 뒤섞인 제비 중 하나를 사용자가 직접 선택 → 결과 공개
3. **복권 긁기:** 스크래치 인터랙션으로 가려진 영역을 긁어 결과 확인
4. **룰렛:** 후보 메뉴가 배치된 원판을 돌려 정지 위치의 메뉴 선택
5. **슬롯머신:** 레버/버튼 조작으로 릴 회전 → 정지 시 표시된 메뉴 선택

## 6. 사용자 흐름 (User Flow)

```
접속 → 키워드 검색 → 음식점 목록 탐색 → 음식점 선택
     → 전체 메뉴 확인 → 후보 메뉴 구성(추가/제외)
     → 게임 선택 → 게임 참여 → 결과(선택된 메뉴) 확인 → 전체 메뉴 확인
```

## 8. 범위 (Scope)

### 8.1 포함 (In Scope) — MVP
- 키워드 기반 음식점 검색 및 목록/상세 조회
- 후보 메뉴 구성 (추가/제외/전체 선택·해제)
- 5종 랜덤 게임 및 결과 표시

### 8.2 제외 (Out of Scope) — 추후 검토
- 회원가입/로그인, 개인화 추천
- 리뷰·평점, 주문/예약 연동
- 결과 공유(SNS), 다인 참여(멀티플레이) 게임
- 위치 기반 자동 검색
